# 替换为 Skill 名称

## 用途

这是后续创建项目级技术栈 skill 的模板。

不要把这个文件视为已经生效的技术栈规则。只有在技术栈确定后，才能复制本模板并创建具体 skill。

## 技术栈 Skill 必须包含的内容

一个完整的技术栈 skill 必须定义：

- 什么时候必须加载该 skill。
- 文件与目录组织规则。
- 拆分文件、模块或组件的判断标准。
- 该技术栈下的测试要求。
- 本项目中特别需要注意的常见问题。
- 收集证据和验证结果时应使用的命令或检查方式。

## 状态

仅占位。不是已批准的项目规则。
