# Go 错误设计

## 核心原则

错误是 API 契约的一部分，但大多数错误不需要成为新的类型。默认返回标准 `error`，在边界添加有意义的上下文；只有调用方确实需要稳定分支处理时，才暴露 sentinel 或类型错误。

不要为小型服务预先创建通用 `fault` package、`business/dependency/internal` 分类树或万能错误码体系。先从实际调用方和失败处理需求收集证据。

需要完整示例时，阅读 `references/error-examples.md`；需要来源依据时，阅读 `references/source-notes.md`。

## 创建与包装错误

- 使用 `errors.New` 创建没有动态数据的简单错误。
- 使用 `fmt.Errorf("operation: %w", err)` 添加上下文并保留需要公开的错误身份。
- `%w` 会让底层错误成为可由调用方检查的契约；不希望暴露时使用 `%v` 或在边界转换为新的稳定错误。
- adapter 需要隔离 SDK 类型时，不要直接用 `%w` 把 SDK error 返回上层；先保留取消/超时语义，再转换为本地稳定错误和安全诊断上下文。
- 错误消息使用小写开头且不加句末标点，避免多层包装后出现破碎句子。
- 不丢弃 `context.Canceled` 或 `context.DeadlineExceeded` 的 `errors.Is` 判定语义。

## 判定错误

- 使用 `errors.Is` 判断 sentinel 或具备等价语义的错误。
- 使用 `errors.As` 提取调用方需要读取的类型错误。
- 禁止通过 `err.Error()`、字符串包含或完整文本相等来控制程序分支。
- 测试错误身份和公开字段，不把完整人类可读消息当成稳定协议，除非消息本身明确属于产品契约。

## Sentinel 与类型错误

| 需求 | 默认选择 |
| --- | --- |
| 调用方只需判断一种稳定条件 | package 级 sentinel，例如 `ErrNotFound` |
| 调用方还需读取结构化数据 | 聚焦的类型错误，并按需实现 `Unwrap` |
| 失败只用于诊断，不需要调用方分支 | 普通 `error` 加上下文 |
| 底层 SDK 错误不属于公开契约 | 在 adapter 或应用边界转换，不直接泄漏 |

导出 sentinel 或类型后，名称、身份和可判定语义都具有兼容性成本。不要导出每一种内部失败，也不要把第三方 SDK 的具体错误类型变成领域 package 的 API。

## Package 与协议边界

- 领域 package 返回领域调用方能理解的错误，不导入 `net/http` 来表达状态码。
- adapter 可以包装底层错误以便诊断，但向上层只暴露所需的稳定语义。
- HTTP、RPC、CLI 等入口集中把已知错误映射为协议响应；未知错误使用通用安全响应。
- 不把 `err.Error()`、堆栈、内部地址、令牌、凭证或个人敏感数据直接返回客户端；日志也必须按数据敏感度脱敏。
- 在应用或协议边界记录一次错误；底层负责包装上下文，不在每一层重复日志。

## 重试与 panic

- “是否可重试”取决于具体操作的幂等性、失败阶段和调用语义，不默认放进通用错误基类。
- 超时不自动等于可重试；先判断操作是否可能已经生效。
- `panic` 只用于不可恢复的不变量或程序员错误，不用于可预期的输入、网络、配置或依赖失败。

## 文件位置

- 错误只服务一个小 package 时，放在相关实现附近。
- package 有多个调用方共享的稳定错误契约时，可以使用聚焦的 `errors.go`。
- 不创建跨领域通用错误 package，除非已有多个真实调用方共享相同语义，而不只是共享字段形状。

## 验证

优先沿用仓库已有命令；没有项目命令时，从受影响 module 根目录运行：

```text
go test ./...
go vet ./...
```

测试至少覆盖公开 sentinel/type 的 `errors.Is/As`、多层包装、取消与超时语义、协议映射和未知错误不泄漏内部信息。

## 常见错误

| 错误 | 处理 |
| --- | --- |
| 比较 `err.Error()` 文本 | 改用 `errors.Is/As` 或明确的公开契约 |
| 每种失败都定义错误码和分类对象 | 只保留调用方实际处理的稳定差异 |
| 领域错误携带 HTTP status | 在 HTTP 边界集中映射 |
| 每层既 wrap 又 log | 底层 wrap，边界统一记录一次 |
| 直接返回 SDK 错误或内部地址 | 在边界转换并返回安全消息 |
| 把 timeout 一律标为可重试 | 结合幂等性和失败阶段判断 |
